推荐方案：拆分主按钮（Split Button）

  • 主区域（珊瑚色，⌘N）：调用 getCurrentDaemonAgent(teamId)，用 createSessionShell 创建仅含「你 + 本机 Agent」的会话，setEngagedAgent 预选 Agent，不弹窗，聚焦输入框。
  • ▾ 菜单项「多人会话…」：打开现有 NewSessionDialog（选人 + 可选开场白），行为与今天一致。
  • 标题：沿用 ChatPanel 逻辑 AgentName (HH:mm)，便于同 Agent 多会话区分。

与现状对比

现在

新聊天 / ⌘N → NewSessionDialog（必选参与者） → 必填开场消息 → 创建

目标

新聊天 / ⌘N → createSessionShell([me, myAmuxd]) → 空线程 + 输入框聚焦 多人会话… → NewSessionDialog（不变）

其它方案（未采用）

  • 双按钮纵向堆叠：「与 amuxd 聊」+ 次要链接「多人…」— 占用侧栏高度，弱于拆分按钮的一体感。
  • 零会话壳、仅切空白 ChatPanel：与列表/实时订阅模型不一致，需更大重构。

边界与实现备注（供后续开发）

  • 本机 Agent 解析：已有 getCurrentDaemonAgent（device_id / is_owner 回退）。
  • 无 Agent：禁用主按钮 + 链到 Daemon onboarding / 设置；多人入口仍可尝试（团队目录）。
  • ChatPanel 带正文触发新建：若已 @ 多人，仍走 Dialog；仅「单 Agent 且为本机」时可快路径。
  • Web 非 Tauri：无 device 时回退 owner Agent 或提示绑定。
  • i18n 建议键：chat.newChatWithDaemon、chat.newMultiPersonSession。
