空会话 Thread 占位 — In-Session Empty State

场景：已有 activeSessionId、消息列表为空、底部 Composer 已显示。 与「无会话待命页」不同——不再创建会话，不再出现「开始对话」主按钮； 中间区域应像 thread 里的第一条 Agent 便签，引导用户在下方输入或点 chip 预填 Composer。

与待命页差异

  • 无「开始对话」CTA（已在会话内）
  • 无 SessionContinueBanner（续聊在待命页）
  • chip → 预填 / 发送 Composer，不 createSession
  • 可展示 model、workspace 等会话上下文

视觉原则

  • 复用 Agent 回复 memo 结构（AGENTS.md §3）
  • 内容贴 thread 底部（justify-end）
  • coral 仅用于 AI pill / avatar ring
  • 首条真实消息后整块消失

实现锚点

  • ChatPanel emptyState when activeSessionId
  • 新组件 SessionEmptyThreadState
  • chip onClick → requestComposerFocus + setInputValue

推荐 B：像 Agent 已经发来一条「开场 memo」——avatar + model 元信息 + 简短能力说明 + dashed 能力网格 + follow-up chips。 与有消息时的 note 视觉连续，空 thread 不再是大片留白。A 偏卡片居中，_session 感弱；C 太淡。
