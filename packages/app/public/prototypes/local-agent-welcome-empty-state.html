本地 Agent 待命页 — Empty State 设计（v2 · 无输入框）

无选中会话时，中间区域不显示 Composer 输入框，只展示本机 agent 待命态。 用户通过「开始对话」按钮、快捷 chip或续聊链接进入会话——行为与左上角 New Chat（createQuickDaemonSession）一致。 进入会话后才出现底部输入框。

触发方式

  • 开始对话 → 创建空会话 + focus composer
  • 快捷 chip → 创建会话 + 发送首条消息
  • 继续上次 → switchToSession
  • ⌘N → 同「开始对话」（sidebar 已有）

与 v1 差异

  • 移除底部 Composer（无 session 时不渲染）
  • 去掉「输入任意内容即可开始」类文案
  • 主 CTA 替代输入框成为入口

边界状态

  • 在线：CTA + chip 可点
  • 离线：CTA 禁用 + 跳转设置
  • 群聊仍走 New Chat ▾ → 多人会话

推荐方案 B：Agent 便签头 + 一句待命说明 + 「开始对话」主按钮 + 可选 chip。 页面底部刻意留白——没有输入框，避免「还没创建会话就能打字」的误导。
